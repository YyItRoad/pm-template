# 模板 [Phase 1] 需求文档

> 用途: 锁住"做什么、不做什么、做到什么程度算完"
> 输入: 来自 Phase 0 charter
> 产出: 用户故事 + 验收标准 + 边界场景,所有 phase 2+ 的源头
> 禁止: 不写架构、不写接口、不写表 — 那是 Phase 2/3 的事

## §0 挖掘证据(强制,不可跳过)

> 本节是 §1-§6 的**结构化摘要**,所有 anchor 必填,数量不达标 → critic CRITICAL。
