# 模板 [Change] Refactor — 重构(不改行为)

> 用途:改代码结构、改命名、提抽象、改文件组织——**对外行为完全不变**。
> 严格度:**中**(7 项 critic / 7 项 DoD / 1 人 + 1 reviewer 签字)。
> 禁止:借 refactor 修 bug——必须先开 bugfix;借 refactor 加能力——必须先开 feature。

---

  - 可测试:___
  - 复用:___
- **目标**(用度量说话,不要"让代码更清晰"):

## §1.2 范围 [必填]

- **涉及文件**:`path/to/file.py` 列表
- **预计 LOC 变化**:+X / -Y(允许 ±,重点是**行为不变**)
- **不动的边界**:哪些模块/接口**不能动**?

## §1.3 行为不变证明 [必填,★关键]

> 这是 refactor 和 bugfix/feature 的**唯一区别**——必须证明行为 0 变化。

- **现有 e2e 通过**(列出 e2e 脚本 + 通过 case 数):___
- **新增对比测试**(如有):___

- **数据库兼容性**:无 schema 变化?Y / N(否则升级到 migration 类变更)
