# DoD [Phase 1] 需求文档

> 全部勾上才允许进 Phase 2。

- [ ] 1.1 角色清单 ≥2 行,每行 4 列都填
- [ ] 2.1 总故事数 ≥8 且 ≤30(太少=没想清楚,太多=没拆)
- [ ] 2.2 每条故事有 S-ID + 来源 + ≥1 AC
- [ ] 2.3 每条 AC 可机械验证(grep 查"差不多/基本/友好/快速"这类模糊词命中 = 0)
- [ ] 2.4 每条故事的 AC 数量 ≤3(超过必拆)
- [ ] 3.1 边界场景 ≥5 条
- [ ] 3.2 边界场景覆盖:并发 / 鉴权失败 / 数据不存在 / 状态非法(各 ≥1 条)
- [ ] 4.1 非功能需求 4 项(性能/安全/合规/可观测)填了(可写"N/A"但不能空白)
- [ ] 5.1 反向需求 ≥3 条
- [ ] 6.1 决策记录 ≥1 条
- [ ] 7.1 **critic 自审报告无 CRITICAL/HIGH**(报告路径填了)
